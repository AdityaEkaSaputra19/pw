Pemrograman Web
Modul DOM & JavaScript Vanilla

1. Web Programming Stack
Point: Web programming dibangun di atas tiga teknologi utama yang bekerja bersama yaitu HTML, CSS, dan JavaScript.
Reason: Ketiganya punya peran yang berbeda dan saling melengkapi. HTML membentuk struktur konten, CSS mengatur tampilan, dan JavaScript menambahkan interaktivitas. Tidak ada yang bisa menggantikan peran satu sama lain.
Example: Bayangkan membangun sebuah rumah. HTML adalah fondasi dan dinding rumahnya, CSS adalah cat, furnitur, dan dekorasinya, sedangkan JavaScript adalah sistem listrik dan kunci pintu otomatis yang membuat rumah itu bisa bereaksi terhadap penghuninya.
Point: Maka dari itu, ketiga teknologi ini harus dipelajari dan dipahami secara beriringan karena di dunia nyata ketiganya selalu hadir bersama dalam satu proyek web.
2. DOM (Document Object Model)
Point: DOM adalah representasi struktur halaman HTML yang dibangun oleh browser berbentuk pohon bercabang, dan inilah yang bisa dimanipulasi oleh JavaScript.
Reason: Ketika browser membaca file HTML, ia tidak hanya menampilkannya — ia membangun sebuah model objek dari seluruh isi halaman di dalam memori. Model inilah yang menjadi pintu masuk bagi JavaScript untuk membaca, mengubah, menambah, atau menghapus elemen secara langsung tanpa perlu memuat ulang halaman.
Example: Bayangkan sebuah dokumen HTML seperti buku. DOM adalah daftar isinya yang sudah terstruktur rapi — setiap bab dan paragraf tercatat dan bisa langsung diakses. JavaScript bisa membuka halaman mana saja, mengganti isinya, atau menambah halaman baru tanpa harus mencetak ulang seluruh buku dari awal.
Point: Jadi DOM adalah jembatan utama antara HTML yang statis dan JavaScript yang dinamis — tanpa DOM, JavaScript tidak punya cara untuk menyentuh apapun yang ada di halaman.
3. DOM Selector
Point: Sebelum JavaScript bisa mengubah sesuatu, ia harus terlebih dahulu memilih elemen mana yang ingin diubah, dan itulah fungsi dari DOM Selector.
Reason: Halaman web bisa memiliki ratusan elemen sekaligus. JavaScript butuh cara yang presisi untuk menunjuk tepat ke elemen yang dimaksud. Ada dua pendekatan utama yaitu getElementById yang hanya mencari berdasarkan ID unik, dan querySelector yang jauh lebih fleksibel karena bisa mencari berdasarkan kelas, ID, tag, bahkan kombinasi elemen bertingkat sekaligus.
Example: getElementById ibarat mencari buku di perpustakaan berdasarkan nomor ISBN-nya yang unik dan pasti. Sedangkan querySelector ibarat mencari buku berdasarkan kategori, nama penulis, atau warna sampulnya — lebih fleksibel dan cocok untuk berbagai situasi.
Point: Untuk kode yang modern dan mudah dikembangkan, querySelector adalah pilihan yang lebih direkomendasikan karena kemampuannya yang jauh lebih luas dan sintaksnya yang mirip dengan CSS yang sudah familiar.
4. Event Listener
Point: Event listener adalah mekanisme yang membuat JavaScript mampu mendengarkan aksi pengguna dan meresponsnya secara otomatis.
Reason: Interaktivitas adalah inti dari sebuah web aplikasi. Tanpa event listener, halaman hanya diam seperti gambar — tidak bisa merespons klik, ketikan, maupun gerakan apapun dari pengguna. Event listener menghubungkan antara tindakan pengguna dengan kode yang harus dijalankan.
Example: Bayangkan seorang resepsionis hotel yang selalu siaga. Event listener adalah resepsionis itu — ia menunggu tamu melakukan sesuatu. Ketika tamu menekan bel (klik tombol), resepsionis langsung bereaksi menjalankan tugasnya. Jenis kejadian yang bisa didengarkan antara lain klik, perubahan nilai input, hingga pengiriman formulir.
Point: Maka pola dasarnya selalu sama di hampir semua fitur interaktif web yaitu pilih elemen, pasang event listener, lalu tentukan fungsi yang akan dijalankan ketika kejadian itu terjadi.
5. Callback & Fungsi
Point: Callback adalah fungsi yang disiapkan untuk dijalankan nanti ketika suatu kejadian terjadi, bukan langsung dieksekusi saat kode dibaca.
Reason: Ini yang membuat JavaScript unik. Ia tidak harus menunggu satu tugas selesai sebelum bisa bersiap untuk tugas lain. Callback memungkinkan JavaScript menyiapkan instruksi di awal, lalu mengeksekusinya hanya ketika kondisi atau kejadian yang tepat datang.
Example: Bayangkan kamu menitipkan pesan kepada teman: Kalau ada orang meneleponku, tolong beritahu mereka aku akan segera balik. Pesan itu adalah callback — kamu tidak tahu kapan orang itu menelepon, tapi instruksinya sudah siap dan akan dijalankan tepat saat kejadian itu terjadi.
Point: Memahami callback adalah kunci untuk memahami hampir semua fitur JavaScript yang lebih lanjut, mulai dari event handling, pengambilan data dari server, hingga animasi.
6. Output & Debugging
Point: Untuk melihat hasil kerja kode atau menemukan kesalahan, JavaScript menyediakan dua alat utama yaitu alert untuk pop-up dan console untuk pencatatan diam-diam di belakang layar.
Reason: Programmer tidak bisa menulis kode secara buta tanpa bisa melihat apa yang sedang terjadi. Kedua alat ini adalah jendela untuk mengintip kondisi program, sangat penting saat belajar maupun saat mencari bug di kode yang sudah kompleks.
Example: alert ibarat pengumuman lewat pengeras suara yang langsung muncul dan menghentikan segalanya — cocok untuk demonstrasi cepat tapi mengganggu jika dipakai terus-menerus. console.log ibarat buku catatan harian yang mencatat semua kejadian secara diam-diam dan hanya bisa dilihat jika kamu membuka panel Developer Tools di browser (tekan F12).
Point: Biasakan menggunakan console.log daripada alert karena lebih profesional, tidak mengganggu pengguna, dan memberikan informasi yang jauh lebih kaya dan detail untuk proses debugging.
7. Vanilla JS vs Framework
Point: Vanilla JS adalah JavaScript murni tanpa tambahan apapun, dan inilah yang harus dikuasai sebelum melangkah ke framework seperti React atau Vue.
Reason: Framework hanyalah lapisan abstraksi yang dibangun di atas JavaScript murni. Jika fondasi vanilla JS-nya lemah, belajar framework akan terasa seperti menghafal mantra tanpa memahami maknanya — bisa digunakan tapi tidak bisa dipecahkan masalahnya ketika ada yang salah.
Example: Vanilla JS seperti belajar memasak dari nol — memotong sendiri, mengatur api sendiri, mengukur bumbu sendiri. Framework seperti menggunakan bumbu instan dan peralatan canggih — prosesnya lebih cepat, tapi kalau tidak paham dasar memasaknya, hasilnya bisa kacau.
Point: Kuasai vanilla JS dengan benar terlebih dahulu. Setelah DOM, event, selector, dan callback benar-benar dipahami secara mendalam, belajar framework apapun akan terasa jauh lebih mudah, logis, dan menyenangkan.
