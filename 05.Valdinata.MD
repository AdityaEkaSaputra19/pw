Pemrograman Web — Modul DOM & JavaScript Vanilla (Framework PREP)

PREP 1 — Web Programming: Gambaran Besar
Point: Web programming modern dibangun di atas tiga teknologi utama yang bekerja bersama-sama.
Reason: Setiap teknologi punya peran yang berbeda dan saling melengkapi. Tidak bisa digantikan satu sama lain karena fungsinya berbeda secara mendasar.
Example: Bayangkan membangun sebuah rumah. HTML adalah fondasi dan dinding rumahnya, CSS adalah cat, furnitur, dan dekorasi yang membuatnya indah, sedangkan JavaScript adalah sistem listrik, kunci pintu otomatis, dan semua hal yang membuat rumah itu bisa "bergerak" dan merespons penghuninya.
Point (ulang): Ketiga teknologi ini wajib dikuasai secara bersamaan karena di dunia nyata ketiganya selalu digunakan bersama dalam satu proyek.

PREP 2 — DOM: Apa dan Mengapa Penting?
Point: DOM (Document Object Model) adalah representasi dari halaman HTML yang dibuat oleh browser, berbentuk seperti pohon yang bercabang-cabang.
Reason: Ketika browser membaca file HTML, ia tidak hanya menampilkannya begitu saja — ia membangun sebuah "model" dari halaman tersebut di dalam memori komputer. Model inilah yang bisa disentuh, diubah, dan dimanipulasi oleh JavaScript secara langsung tanpa perlu memuat ulang halaman.
Example: Bayangkan sebuah dokumen HTML seperti sebuah buku. DOM adalah daftar isi buku itu yang sudah terstruktur rapi — setiap bab, sub-bab, dan paragraf tercatat dengan jelas dan bisa langsung diakses. JavaScript bisa membuka halaman mana saja, mengganti isi tulisannya, atau bahkan menambah halaman baru tanpa harus mencetak ulang seluruh buku.
Point (ulang): DOM adalah jembatan antara HTML yang statis dan JavaScript yang dinamis. Tanpa DOM, JavaScript tidak punya "pintu masuk" ke dalam halaman web.

PREP 3 — DOM Selector: Cara Memilih Elemen
Point: Sebelum JavaScript bisa mengubah sesuatu di halaman, ia harus terlebih dahulu "memilih" elemen mana yang ingin diubah — inilah fungsi DOM Selector.
Reason: Halaman web bisa memiliki ratusan bahkan ribuan elemen HTML. JavaScript perlu cara yang presisi untuk menunjuk tepat ke elemen yang dimaksud, seperti kamu memilih satu buku dari rak perpustakaan yang penuh.
Example: Ada dua pendekatan utama. Pertama, getElementById — cara yang sangat spesifik, hanya bisa mencari berdasarkan ID unik sebuah elemen, ibarat mencari buku berdasarkan nomor ISBN-nya. Kedua, querySelector — cara yang jauh lebih fleksibel, bisa mencari berdasarkan nama kelas, ID, jenis tag, bahkan kombinasi ketiganya sekaligus, ibarat mencari buku berdasarkan kategori, penulis, atau warna sampulnya.
Point (ulang): Untuk kode yang modern dan fleksibel, querySelector adalah pilihan yang lebih direkomendasikan karena kemampuannya yang jauh lebih luas.

PREP 4 — Event Listener: Membuat Halaman Bereaksi
Point: Event listener adalah mekanisme yang memungkinkan JavaScript "mendengarkan" aksi yang dilakukan pengguna, lalu meresponsnya secara otomatis.
Reason: Interaktivitas adalah jiwa dari sebuah web aplikasi. Tanpa event listener, halaman hanya diam seperti gambar — tidak bisa merespons apapun yang dilakukan pengguna, baik itu klik, ketikan, maupun gerakan mouse.
Example: Bayangkan seorang resepsionis hotel. Event listener adalah resepsionis itu — ia selalu siaga menunggu tamu (pengguna) melakukan sesuatu. Ketika tamu menekan bel (klik tombol), resepsionis langsung bereaksi menjalankan tugasnya (fungsi callback). Jenis kejadian yang bisa "didengarkan" antara lain klik tombol, perubahan nilai pada kolom input, hingga pengiriman formulir.
Point (ulang): Pola dasar yang selalu dipakai adalah: pilih elemen → pasang event listener → tentukan fungsi yang akan dijalankan. Pola ini berulang di hampir semua fitur interaktif web.

PREP 5 — Fungsi & Callback
Point: Callback adalah fungsi yang disiapkan untuk dijalankan nanti, bukan sekarang — ia dijalankan hanya ketika sebuah kejadian tertentu terjadi.
Reason: Ini adalah konsep yang membedakan JavaScript dari banyak bahasa pemrograman lain. JavaScript tidak harus menunggu satu tugas selesai sebelum mengerjakan tugas lain — ia bisa menyiapkan instruksi yang akan dieksekusi di masa depan ketika kondisinya tepat.
Example: Bayangkan kamu menitipkan pesan kepada teman: "Kalau ada orang meneleponku, tolong beritahu mereka aku akan segera balik." Pesan itu adalah callback — kamu tidak tahu kapan orang itu akan menelepon, tapi instruksinya sudah siap dan akan dijalankan tepat saat kejadian itu terjadi.
Point (ulang): Memahami callback adalah kunci untuk memahami bagaimana JavaScript mengelola interaktivitas, permintaan data ke server, dan semua hal yang bersifat "menunggu" sebelum dijalankan.

PREP 6 — Output & Debugging: Alert dan Console
Point: Untuk melihat hasil kerja kode atau mencari kesalahan, JavaScript menyediakan dua alat utama yaitu alert untuk pop-up dan console untuk pencatatan di belakang layar.
Reason: Seorang programmer tidak bisa menulis kode secara buta tanpa bisa melihat apa yang terjadi. Kedua alat ini adalah "jendela" yang memperlihatkan apa yang sedang terjadi di dalam kode, sangat penting terutama saat belajar dan saat mencari bug.
Example: alert ibarat pengumuman lewat pengeras suara yang langsung muncul dan menghentikan segalanya sampai diklik OK — cocok untuk demonstrasi cepat tapi mengganggu jika dipakai terus-menerus. console.log ibarat buku catatan harian yang mencatat semua kejadian secara diam-diam di balik layar — hanya bisa dilihat jika kamu membuka panel Developer Tools di browser (tekan F12), sangat ideal untuk debugging serius.
Point (ulang): Biasakan menggunakan console.log daripada alert untuk debugging karena lebih profesional, tidak mengganggu pengalaman pengguna, dan memberikan informasi yang jauh lebih detail.

PREP 7 — Vanilla JS vs Framework
Point: Vanilla JS berarti menggunakan JavaScript murni tanpa tambahan library atau framework apapun, sedangkan framework seperti React atau Vue adalah alat bantu yang dibangun di atas JavaScript murni itu sendiri.
Reason: Banyak pemula tergoda langsung belajar React atau Vue, padahal framework tersebut hanyalah lapisan abstraksi di atas JavaScript murni. Jika fondasi vanilla JS-nya lemah, belajar framework akan terasa seperti menghafal mantra tanpa memahami artinya.
Example: Vanilla JS adalah seperti belajar memasak dari awal — memotong sayuran, mengatur api, mengukur bumbu sendiri. Framework adalah seperti menggunakan bumbu instan dan alat masak modern — prosesnya lebih cepat, tapi kalau tidak paham dasar memasaknya, hasilnya bisa kacau dan kamu tidak akan bisa menyelesaikan masalah ketika sesuatu tidak berjalan sesuai rencana.
Point (ulang): Kuasai vanilla JS terlebih dahulu dengan benar. Setelah paham DOM, event, selector, dan callback secara mendalam, belajar framework apapun akan terasa jauh lebih mudah dan masuk akal.

Alur Belajar yang Tepat (Sesuai Whiteboard)
Berdasarkan catatan di papan tulis, urutan yang ideal adalah pertama memahami struktur dasar web (HTML + CSS + JS), kemudian masuk ke konsep DOM dan cara memilih elemen, lalu belajar event listener dan callback, kemudian berlatih mengambil dan memanipulasi data/nilai dari elemen, dan terakhir baru beranjak ke framework setelah fondasi vanilla JS benar-benar kuat.
